버퍼 : 1. 일정한 크기로 모아두는 데이터
       2. 일정한 크기로 채우고 보내는 타입(10MB라고 치면 10MB모으고 전달) 
버퍼링 : 로딩 시 버퍼링 생기는게 일정 데이터가 모아져야 실행이 되는데 모아지지 않아서 발생
스트림 : 1. 데이터의 흐름(일정한 크기로 나눠서 여러번에 걸쳐 처리)
         2. 작은 크기의 데이터를 지속적으로 전달하는 타입 
스트리밍 : 일정한 크기의 데이터(버퍼)를 지속적으로 전달하는 작업
         : 파일 다운로드 할 때 조금씩 차는 것

버퍼와 스트림 중에 스트림이 효율적
ㄴ 스트림 방식을 해야 서버에 메모리 적게 차지하면서 효율적으로 데이터를 보낼 수 있음

파이프(pipe) : 스트림 방식으로 쪼개져 있는 데이터들을 연결할 수 있음, 원하는 순서대로 조작 가능함

파일 압축(zlib) : 파일을 압축할 수 있음
